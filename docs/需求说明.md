开发一套功能完整、界面现代化的家族族谱展示网站，实现从本地Excel表格到动态Web应用的转换，确保在移动端和PC端均有良好的展示效果。该项目需包含前端静态展示页面和后台维护管理界面两部分，具体技术要求和功能规范如下：

### 技术栈与架构要求

1. 后端开发：使用.NET 10框架进行开发，采用分层架构（MVC架构）设计（建议包含表示层、业务逻辑层、数据访问层），做到前后端分离。
2. 前端开发：建议使用Asp.NET Core 11.0框架进行开发，确保在移动端和PC端均有良好的展示效果。
3. 数据访问：使用SqlSugar.Core库作为ORM框架
4. 数据库选择：采用MySQL作为数据库解决方案（考虑到与.NET生态的兼容性及部署便捷性）
5. 部署方式：支持Docker容器化部署，需提供完整的Dockerfile及docker-compose配置文件
6. 其他要求：要求网站现代化、简约素净、功能齐全、响应式设计（适配移动端和PC端）、且符合中国网络规范（6.1.1.1-6.1.1.255）

### 功能模块详细规范

#### 1. 前端展示系统

1.1 首页设计

- 页面上部：展示家族介绍、族谱意义等文字说明内容，支持富文本格式
- 页面中部：以卡片或列表形式展示多本家谱，每个家谱项需包含家谱名称、简介、创建日期等基本信息
- 页面下部：展示网站备案信息、更新日期、版权声明及其他注释内容，样式需与整体设计协调

1.2 家谱访问控制

- 实现家谱访问验证机制：每个家谱可配置独立的验证问题（Key-Value形式）
- 验证规则：答案包含指定关键词即可通过验证
- 访问权限设置：支持配置是否需要验证（后台可切换"需要验证"/"直接访问"模式）
- 家谱与验证问题为一对多关系，一个家谱可配置多个验证问题（按顺序验证）

1.3 家谱展示功能

- 提供多种展示形式，包括但不限于：
  - 吊线图展示：以树状结构展示家族成员间的层级关系
  - 左右、上下世系表格图展示：以表格形式展示家族成员间的层级关系 
  - 树形结构展示：直观展示家族成员间的层级关系
  - 表格形式展示：详细列出家族成员信息，支持排序和筛选
  - 时间轴展示：按时间顺序展示家族重要事件和人物
  - 关系图谱展示：以图形化方式展示家族成员间的复杂关系
- 所有展示形式需支持响应式设计，在不同设备上均有良好表现
- 支持成员信息查看：点击成员节点可查看详细信息（包括基本信息、生平事迹、照片等）

#### 2. 后台管理系统

2.1 管理员功能

- 用户管理：支持管理员账户的创建、编辑、删除及权限分配
- 登录认证：实现安全的登录机制，支持密码加密存储
- 操作日志：记录管理员的关键操作，便于审计和问题追踪

2.2 家谱维护功能

- 家谱信息管理：支持家谱的创建、编辑、删除和状态管理
- Excel导入功能：实现从Excel表格批量导入家谱数据，支持数据校验和错误提示
- 成员管理：支持单个或批量添加、编辑、删除家族成员信息
- 多媒体管理：支持上传和管理家族成员照片、家族相关文档等
- 家谱成员关联：支持将成员与家谱关联，实现成员的分类展示

2.3 相册维护功能
- 相册管理：支持创建、编辑、删除相册
- 相册照片上传：支持上传和管理相册中的照片
- 成员照片：成员的个人照片，用于展示和识别
- 相册照片展示：在相册展示时，支持点击缩略图查看大图
- 相册照片删除：支持删除相册中的照片

2.4 数据字段说明
- 家谱表包含且不限于以下字段：
  - 家谱ID：GUID类型，唯一标识符，用于区分不同的家谱
  - 家谱名称：家谱的名称，用于展示和识别
  - 家谱简介：家谱的简单介绍，用于展示和筛选
  - 创建日期：家谱创建的时间，用于记录和排序
  - 家谱关联：将成员与特定家谱关联，实现成员的分类展示

- 成员表包含且不限于以下字段：
  - 世代：成员所属的世代，用于展示和筛选（自动计算）
  - 姓氏：成员的姓氏（某些成员尚不明确具体姓名，只知道姓氏）
  - 字号别称：成员的字号别称（如"张三"、"李四"等）
  - 排行：成员在世代中的排名，用于展示和筛选（如"第一"、"第二"等）
  - 字辈：同胞同族兄弟有可能是一个字辈，用于展示和筛选
  - 生辰公历：成员的公历出生日期，可根据生辰农历自动计算
  - 生辰农历：成员的农历出生日期，可根据生辰公历自动计算
  - 居住地：成员的居住地
  - 职业：成员的职业（如"医生"、"律师"等）
  - 个人信息：成员的个人信息（如"1990年1月1日出生于中国"）
  - 小注：成员的备注信息，用于补充说明或特殊记录（如"该成员是家族的代表"）
  - 卒亡标识：成员是否已死亡，用于展示和筛选
  - 卒亡农历：成员的死亡日期（如果有），可根据卒亡公历自动计算
  - 卒亡公历：成员的死亡日期（如果有），可根据卒亡农历自动计算
  - 备注：成员的备注信息，用于补充说明或特殊记录



2.5 问题维护功能

- 验证问题管理：支持添加、编辑、删除验证问题（Key-Value形式）
- 家谱关联：可将问题与特定家谱关联，配置验证顺序和规则

2.6 系统管理功能

- 数据库状态检查：提供数据库连接状态、表结构完整性、数据量统计等信息
- 系统设置：包括网站基本信息配置、展示样式设置、安全设置等
- 数据备份与恢复：支持手动或自动备份数据库，提供数据恢复功能

### 开发与部署要求

1. 代码规范：遵循.NET开发最佳实践，代码需有完善的注释
2. 安全性：实现防SQL注入、XSS攻击等安全措施，敏感数据需加密存储
3. 性能优化：确保页面加载速度快，数据查询高效
4. Docker部署：
   - 提供Dockerfile文件，包含完整的构建和运行环境配置
   - 创建docker-compose.yml文件，支持一键部署应用和数据库
   - 配置数据持久化方案，确保容器重启后数据不丢失

### 交付物

1. 完整的源代码（前端和后端）
2. 数据库脚本（包含表结构和初始数据）
3. Docker相关配置文件（Dockerfile、docker-compose.yml等）
4. 系统部署文档和使用手册

项目开发需确保代码可维护性和扩展性，便于未来功能升级和内容更新。
