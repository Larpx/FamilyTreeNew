# FamilyTreeNew 功能补充计划

## 一、项目现状分析

### 1.1 当前项目功能概览

根据对项目源码的分析，当前 FamilyTreeNew 项目已实现以下核心功能：

| 功能模块 | 已实现功能 | 状态 |
|---------|-----------|------|
| **用户认证** | JWT认证、密码加密、登录限制 | ✅ 已实现 |
| **家谱管理** | 创建、编辑、删除、状态管理 | ✅ 已实现 |
| **成员管理** | 添加、编辑、删除、父子关系 | ✅ 已实现 |
| **相册管理** | 相册CRUD、照片上传 | ✅ 已实现 |
| **验证机制** | 验证问题管理、关键词匹配 | ✅ 已实现 |
| **数据导入** | Excel批量导入 | ✅ 已实现 |
| **系统管理** | 备份恢复、数据库状态检查 | ✅ 已实现 |
| **操作日志** | 基础日志记录 | ✅ 已实现 |

### 1.2 参考项目功能对比

通过调研优秀开源家谱项目（Gramps、MyHeritage、FamilySearch）和企业级.NET管理系统，发现以下关键功能差距：

| 功能类别 | 参考项目功能 | 当前项目状态 | 优先级 |
|---------|-------------|-------------|--------|
| **关系模型** | 家庭单元、配偶关系、收养关系 | ❌ 仅父子关系 | 🔴 高 |
| **事件管理** | 出生、婚姻、死亡、自定义事件 | ❌ 无 | 🟡 中 |
| **地点管理** | 地理信息、地图可视化 | ❌ 无 | 🟡 中 |
| **来源管理** | 资料来源引用、文献记录 | ❌ 无 | 🟡 中 |
| **数据交换** | GEDCOM标准格式导入导出 | ❌ 无 | 🟡 中 |
| **图表报告** | 祖先图、后代图、统计报告 | ❌ 基础展示 | 🟡 中 |
| **系统监控** | 健康检查、性能监控、告警 | ❌ 无 | 🟢 低 |

## 二、功能补充计划

### 2.1 第一阶段：核心管理功能完善（高优先级）

#### 2.1.1 家庭关系模型扩展

**目标**：支持复杂家庭关系（配偶、收养、继亲等）

**功能需求**：
- 家庭单元：定义夫妻关系
- 配偶关系：支持一夫多妻/一妻多夫历史家庭
- 收养关系：记录收养子女
- 继亲关系：记录继父/继母关系

**技术实现**：
- 新增实体：`Family`（家庭单元）、`SpousalRelation`（配偶关系）
- 修改实体：`FamilyMember` 添加 `FamilyId` 字段
- 新增服务：`FamilyService`

**文件路径**：
- `src/FamilyTreeNew.Models/Entities/Family.cs`
- `src/FamilyTreeNew.BLL/Services/FamilyService.cs`

---


### 2.2 第二阶段：家谱专业功能增强（中优先级）

#### 2.2.1 事件管理系统

**目标**：支持记录家族成员的重要生活事件

**功能需求**：
- 事件类型：出生、死亡、婚姻、迁徙、毕业、就职等
- 事件关联：关联人物、地点、来源
- 事件时间：支持公历/农历日期

**技术实现**：
- 新增实体：`Event`、`EventType`
- 新增服务：`EventService`
- 新增控制器：`EventsController`

**文件路径**：
- `src/FamilyTreeNew.Models/Entities/Event.cs`
- `src/FamilyTreeNew.Models/Entities/EventType.cs`
- `src/FamilyTreeNew.BLL/Services/EventService.cs`

---

#### 2.2.2 地点管理系统

**目标**：记录事件发生地点、地点信息、坐标信息、居住信息、埋葬地等，并将照片和地点信息关联起来，支持地理可视化（使用免费API实现）

**功能需求**：
- 地点信息：名称、地址、行政区划、坐标信息
- 坐标记录：经纬度存储
- 地点关联：与事件关联
- 地图展示：集成地图组件展示家族迁徙

**技术实现**：
- 新增实体：`Place`
- 新增服务：`PlaceService`
- 集成地图API

**文件路径**：
- `src/FamilyTreeNew.Models/Entities/Place.cs`
- `src/FamilyTreeNew.BLL/Services/PlaceService.cs`

---

#### 2.2.3 来源管理系统

**目标**：记录家谱资料来源，确保数据可信度

**功能需求**：
- 来源类型：书籍、文献、口述、档案、网络等
- 来源引用：记录出处、页码、作者等
- 来源关联：与人物、事件关联

**技术实现**：
- 新增实体：`Source`、`SourceCitation`
- 新增服务：`SourceService`

**文件路径**：
- `src/FamilyTreeNew.Models/Entities/Source.cs`
- `src/FamilyTreeNew.BLL/Services/SourceService.cs`

---

#### 2.2.4 GEDCOM数据交换

**目标**：支持行业标准GEDCOM格式导入导出

**功能需求**：
- GEDCOM导入：解析标准GEDCOM文件
- GEDCOM导出：生成标准GEDCOM文件
- 数据映射：与现有实体模型映射

**技术实现**：
- 新增服务：`GedcomService`
- 新增控制器：`GedcomController`
- 实现GEDCOM解析器和生成器

**文件路径**：
- `src/FamilyTreeNew.BLL/Services/GedcomService.cs`
- `src/FamilyTreeNew.Api/Controllers/GedcomController.cs`

---

### 2.3 第三阶段：高级功能与系统优化（低优先级）

#### 2.3.1 图表报告系统

**目标**：提供多样化的数据展示和报告生成

**功能需求**：
- 祖先图：展示指定成员的祖先谱系
- 后代图：展示指定成员的后代谱系
- 统计报告：家族人数统计、性别比例、年龄分布
- 网页报告：生成可导出的HTML报告

**技术实现**：
- 新增服务：`ReportService`
- 使用D3.js或ECharts实现可视化

**文件路径**：
- `src/FamilyTreeNew.BLL/Services/ReportService.cs`
- `src/FamilyTreeNew.Web/wwwroot/js/charts/`

---

#### 2.3.2 系统监控与健康检查

**目标**：实现系统状态监控和告警

**功能需求**：
- 健康检查：数据库连接、服务状态
- 性能监控：响应时间、请求统计
- 日志管理：日志查询、日志级别配置
- 告警通知：异常告警、邮件通知

**技术实现**：
- 新增控制器：`MonitorController`
- 集成健康检查中间件
- 实现日志查询接口

**文件路径**：
- `src/FamilyTreeNew.Api/Controllers/MonitorController.cs`

---

#### 2.3.3 定时任务系统

**目标**：支持自动化任务执行

**功能需求**：
- 自动备份：定时数据库备份（对接Duplicati备份工具的API）
- 数据清理：清理过期数据
- 统计更新：定时更新统计数据

**技术实现**：
- 使用Quartz.NET实现定时任务
- 新增服务：`ScheduledTaskService`

**文件路径**：
- `src/FamilyTreeNew.BLL/Services/ScheduledTaskService.cs`

## 六、风险评估与应对措施

| 风险点 | 风险等级 | 应对措施 |
|--------|---------|---------|
| GEDCOM格式兼容性 | 高 | 参考GEDCOM 5.5.1标准，支持主流字段映射 |
| 数据迁移影响 | 中 | 提供数据迁移脚本，确保向后兼容 |
| 性能问题（大数据量） | 中 | 实现分页、缓存策略 |
| 前端菜单动态渲染 | 中 | 提供菜单树API，前端根据权限过滤 |

## 七、总结

本计划基于对当前项目的深入分析和对优秀开源家谱项目的参考，提出了三个阶段的功能补充方案：

1. **第一阶段（高优先级）**：完善核心管理功能，家庭关系扩展
2. **第二阶段（中优先级）**：增强家谱专业功能，包括事件管理、地点管理、来源管理、GEDCOM数据交换
3. **第三阶段（低优先级）**：添加高级功能，包括图表报告、系统监控、定时任务

建议按照优先级顺序逐步实现，确保系统的稳定性和可扩展性。
